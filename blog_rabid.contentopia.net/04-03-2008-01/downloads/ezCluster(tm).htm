ezClustertm - Infinite Scalability for any mix of HTTP 1.1 Web Servers.

 

ezClustertm   is a new way to Cluster HTTP 1.1 Web Servers.

 

ezClustertm   turns any mix of HTTP 1.1 Web Servers into a single web server.

 

ezClustertm   provides for redundancy – this improves up-time by eliminating down-time even during normal system maintenance cycles. 

 

Web Servers that are part of an ezClustertm   configuration can be geographically distributed to improve disaster recovery.

 

ezClustertm   can cluster IIS 6.x Web Servers, Apache Web Servers, Tomcat J2EE Application Servers, ColdFusion JRun Servers along with any mix of any kind of server that can process an HTTP 1.1 Request for Content.

 

ezClustertm   uses Round-Robin or Load-Balanced Request Routing Algorithms to spread-out the Content Requests among the whole Cluster. (Additional Load-Balancing Algorithms can easily be developed such as RAM Allocation that favors the Web Servers that use less RAM or that have more RAM available.)

 

ezClustertm   provides a Web Interface one can use to Administer the whole Cluster from any browser the Admin Staff has access to.

 

ezClustertm   is compatible with any CGI Language or Content Delivery System such as ColdFusion, PHP, Perl, JSP, ASP, ASP.Net or any other CGI Language currently available today or in the future.  Any CGI based Language that can access an ODBC or Native SQL Server Data Source can be used.

 

ezClustertm   is compatible with AJAX technologies, in fact we have an in-house developed AJAX Application Framework that leverages ColdFusion MX 7 and ezClustertm   to provide an unparalleled platform for the deployment of high-impact high-powered AJAX Applications.

 

ezClustertm   provides for a single User Session that is accessible to the entire Cluster of HTTP 1.1 Web Servers using a state-of-the-art ColdFusion MX 7 Interface (additional Languages can be added for a nominal development fee upon request).  This is the key element that is missing when one uses a Cisco Local Director for Cluster Traffic Management, for instance.  This is the heart of the ezClustertm   architecture that makes the whole Cluster of Heterogeneous Web Servers into a single Distributed Web Server.  ezClustertm   also allows ColdFusion Developers to use the Session Scope to process User Session State.

 

ezClustertm   is compatible with any version of Windows as long as that version of Windows can run a Production version of Apache 2.x Web Server and ColdFusion MX 7 Standard. 

 

ezClustertm   can run without ColdFusion MX 7 Standard however with limited functionality and no Graphical Reports one could use to perform Cluster Traffic Analysis & Management.

 

ezClustertm   is compatible with any computer system that can run a suitable version of Windows such as any Intel or AMD CPU based system; any mix of 32 or 64-bit CPU is fully compatible with ezClustertm .

 

ezClustertm   is compatible with SQL Server 2005 however it is recommended that a 64-bit version of SQL Server 2005 be used to leverage the additional power of a suitable 64-bit CPU; it is recommended that a minimum of a Dual Core AMD CPU be used however these CPUs are quite inexpensive – additional power can be gained from using a Dual Core AMD FX-60 CPU or a Dual Core AMD Opteron for instance.

 

ezClustertm   can be deployed to work with any Relational Database upon request for a nominal development fee however at this time SQL Server 2005 is the standard database platform of choice.

 

ezClustertm   can be configured to work with a redundant set of SQL Server 2005 servers that are suitable configured for data replication from the primary server to a backup server.  The backup SQL Server 2005 server can be used in case the primary server goes down or is unavailable, for instance, however this requires the code that is deployed to handle this level of processing.

 

ezClustertm   works best when the Web Servers and the Database Servers that are configured to be part of the Cluster are interconnected using inexpensive Gigabit NICs (a minimum of one Gigabit NIC per Server is Recommended however additional NICs can be configured to provide more horsepower for the Cluster).

 

ezClustertm   is low-cost.  A run-time license for the ezClustertm   Traffic Management System and Cluster Traffic Analysis and Management Console is priced well below any competitive product such as the Cisco Local Director, for instance.  ezClustertm   run-time license fees for a cluster of 2 web servers is currently $995.00; additional Web Servers can be added to the Cluster for only $495 per server.  Annual Maintenance Contracts are priced at only $250 per Server regardless of the size of the Cluster being managed by ezClustertm.
